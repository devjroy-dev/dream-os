storefront. Profile Studio's 20-photo requirement is the biggest cliff in vendor
onboarding; "Connect IG, pick your 20" turns an hour of re-uploading into two

## 2. THE CONSTRAINTS, PRICED AT BANKING (so 07's read-first inherits them)
- **API reality:** Basic Display is DEAD (deprecated end-2024). The path is the
  Instagram API with Instagram Login — vendors need PROFESSIONAL
  (business/creator) accounts. Most wedding vendors have them; some won't.
- **MANUAL UPLOAD IS THE PERMANENT FALLBACK, NEVER A WALL.** The IG connect is an
  accelerant on the photo step, not a gate in front of it.
- **IMPORT-AND-MIRROR, NEVER HOTLINK.** IG CDN URLs expire; a storefront that
  rots when a token lapses is F-04.113's silent-failure class at portfolio
  scale. Selected photos copy into the estate's own storage at import; the IG
  connection is a source, never a dependency.
- **APP POSTURE (governance, so the two Meta apps never blur):** content import
  needs App Review and therefore App-LIVE — it rides the MAIN Meta app's review
  cycle beside the Embedded Signup / Tech Provider track (one review buys both).
  The parked 06.5 Growth Suite's second app stays never-App-LIVE; this is NOT it.
- **Photo management** (reorder, replace, remove, the 20-slot grid) is the same
  sitting's surface — import feeds it; it does not depend on import.
